History of the “Cube World Escape” Development


Development history of this game began with the idea to make a 3D game for mobile platforms. We liked this idea because 3D games attract almost everybody with their vividness, game world and atmosphere. We faced the problem of choosing game genre and game scenario. As it was out first experience of game development and considering the lack of time, we decided to choose a puzzle game. We wanted a user to think, but also to get lots of positive emotions and joy each time he makes a right choice.

Today there are already created a lot of various puzzle games. There are real hits among them – “Three in a Row”, «Monument Valley», «The Room» and so on. This only proved our conviction that such kind of games is really popular – people like to play them a lot.


Creation of game plot and game logic


Having chosen genre and idea we started to create a plot. Scenario of a puzzle-game is not very vivid and original and lies in proper distribution of game quests for a player. You need to gradually increase complexity of quests with the progression of the game. After some discussions and looking at the examples of some puzzle-games we decided to make a parallelepiped our main hero. Gameplay itself was in positioning that parallelepiped vertically on a finish platform by successively upturning it through the level map. If a player succeeds – level is passed. If a player falls off the platform the level is failed. There will be a possibility to restart that level or to return to the menu to choose another level (only out of the opened ones) to go. Gameplay turned out to be extremely simple. Main point is complexity and attractiveness of levels, which according to the plan should attract a player on their own by continuously challenging him one level after another. Next task was to organize game logic and mechanics in order to reach the main effect of the game which is to make a player feel comfortable while making a choice and a number of choices and, as a result, combinations of game passing were not limited at all. We wanted to make a game that had only one right solution to complete a level, but a player could have an unlimited choice from a great number of different variants.


Points gaining system


Some discussions took place when choosing a points gaining system. In the process we stopped at the following. In the process player was awarded by a certain number of stars. Game aim is to complete a level with the maximal number of stars. Number of stars awarded for the level completion is counted out of the minimal number of moves (for each level particularly).


3 stars - min =< c < (min + 5%)

2 stars - (min + 6%) < c < (min + 15%)

1 star - (min + 16%) < c < (min + 30%)

0 stars - (min + 31%) < c < infinity

Where:

min - minimal number of moves on this level

c - number of moves made by a player

For example:


On 16 level minimal number of moves (min) equals 63.

In order to get:

3 stars - number of moves made by a player (с) from 63 to 67

2 stars - number of moves made by a player (с) from 68 to 73

1 star - number of moves made by a player (с) from 74 to 82

0 stars - number of moves made by a player (с) from 83 to infinity

Game design


Considering that in this game design plays a crucial role, we decided to regard a design question as one of the most important ones. We required design that could not only emphasize, but also increase player’s impression from the game itself. At the same time it should be clear to a player on a hunch. The first variant of game’s design appeared instantly. The game should be made in old Egyptian style. Or as in films where main characters try to reach the tomb – “Lara Croft”, “Indiana Jones”, etc. While doing this, heroes solve puzzles and look for the keys. The first design looked like this:

After estimating the design we decided that it does not suit us. For this game we need minimalistic design that is not spoiling the general impression. Perhaps, this design did not create the atmosphere according to the initial idea. The second variant of the design became final and it is futuristic. It makes game design and UI design look naturally.
